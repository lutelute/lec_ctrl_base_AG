## 1.1 制御とは何か

### 1.1.1 制御の定義
「制御 (Control)」とは、**ある対象（システム）の状態や出力を、望ましい状態（目標値）に近づける操作**のことです。
私たちの日常生活には、意識せずとも多くの制御が組み込まれています。

- エアコンの室温調節
- 自動車のクルーズコントロール（定速走行）
- 電気ポットの保温機能
- 人間の体温調節（生体制御）

### 1.1.2 制御工学の歴史
制御工学は産業革命とともに発展してきました。

| 年代      | 出来事                                                                   |
| :-------- | :----------------------------------------------------------------------- |
| 1788年    | ジェームズ・ワットの遠心調速機（蒸気機関の回転数制御）                   |
| 1868年    | マクスウェルによる調速機の安定性解析（世界初の安定性論文）               |
| 1932年    | ナイキストの安定判別法                                                   |
| 1940年代  | ボード線図、サーボ機構理論の発展（第二次世界大戦中のレーダー・砲撃制御） |
| 1960年代  | カルマンによる現代制御理論（状態空間法、最適制御）                       |
| 1980年代~ | ロバスト制御、適応制御、非線形制御の発展                                 |

## 1.2 フィードバック制御の基本構造

### 1.2.1 オープンループ制御 vs フィードバック制御
制御には大きく分けて2つの方式があります。

1.  **オープンループ制御（開ループ制御）**:
    - 出力を測定せず、入力のみに基づいて操作を決定する。
    - 例: 洗濯機のタイマー制御（汚れ具合を見ずに時間だけで判断）。
    - **問題点**: モデル誤差や外乱に弱い。

2.  **フィードバック制御（閉ループ制御）**:
    - 出力を測定し、目標値との偏差に基づいて操作を決定する。
    - 例: エアコンのサーモスタット（現在の室温を測定して動作を調整）。
    - **利点**: モデル誤差や予測不能な外乱に対して頑健（ロバスト）。

### 1.2.2 フィードバック制御系のブロック線図
標準的なフィードバック制御系は以下の要素で構成されます。

```
         r      e       u              y
目標値 ---->(+)--->[コントローラC]--->[プラントP]---+---> 出力
             ^                                     |
             |--------[センサH]<-------------------+
                        (フィードバック経路)
```

| 記号   | 名称                               | 役割                          |
| :----- | :--------------------------------- | :---------------------------- |
| $r(t)$ | 目標値 (Reference)                 | 出力にこうなってほしい値      |
| $e(t)$ | 偏差 (Error)                       | $e = r - y$（目標と出力の差） |
| $C(s)$ | コントローラ (Controller)          | 偏差に基づき操作量を決定      |
| $u(t)$ | 操作量 (Control Input)             | アクチュエータへの指令        |
| $P(s)$ | プラント (Plant)                   | 制御対象のシステム            |
| $y(t)$ | 出力 (Output)                      | 実際に観測される量            |
| $H(s)$ | センサ (Sensor / Feedback Element) | 出力を測定しフィードバック    |
| $d(t)$ | 外乱 (Disturbance)                 | 予期せぬ影響（図では省略）    |

## 1.3 制御系設計の8ステップ

制御系を設計する際には、以下の手順を踏みます。

1.  **制御目的の明確化**: 何を、どの程度の精度で制御したいのか？
2.  **制御対象のモデリング**: 物理法則（運動方程式、回路方程式など）から数学モデル（微分方程式、伝達関数）を導出する。
3.  **システム同定（必要に応じて）**: 実験データからモデルのパラメータを推定する。
4.  **制御方式の選定**: PID制御、状態フィードバック、H∞制御など。
5.  **コントローラの設計**: 安定性、過渡特性、定常特性を考慮してパラメータを決定する。
6.  **シミュレーション**: 設計したコントローラをモデル上で検証する。
7.  **実機試験**: 実際の装置で動作を確認し、チューニングを行う。
8.  **評価と改良**: 要求仕様を満たしているか評価し、必要に応じて改良する。

## 1.5 演習問題

**問題**:
以下のシステムについて、フィードバック制御とオープンループ制御のどちらが適切か、理由とともに答えよ。

1.  工場のベルトコンベア（一定速度で製品を運ぶ）
2.  人工衛星の姿勢制御
3.  電子レンジの加熱時間制御

**解答例**:

1.  **ベルトコンベア**: 負荷変動（製品の重さ）や摩擦変化があるため、**フィードバック制御**（速度センサで計測し、モータ出力を調整）が適切。
2.  **人工衛星の姿勢制御**: 宇宙空間には多くの外乱（太陽光圧、微小重力差）があり、かつ精密な角度制御が必要なため、**フィードバック制御**（ジャイロセンサ＋スラスタ）が必須。
3.  **電子レンジの加熱時間**: 食品の種類や量に応じてユーザーが時間を設定するシンプルな方式で十分な場合が多く、**オープンループ制御**が一般的。ただし高級機種では温度センサを用いたフィードバック制御を行う。
